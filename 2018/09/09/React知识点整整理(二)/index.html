React知识点整理（二）

发表于 2018-09-09 | 分类于 前端

对React中的生命周期理解

1.constructor():

存在state，从父级继承props(需要调用super(props))，绑定方法的时候调用。

2.defaultProps:

1.需要在component定义完之后才定义。

2.定义为Component的属性，使用this.props进行调用。

3.initialState:

1.直接定义state，使用this.state.xxx进行调用(es7)。

2.在constructor中使用this.state进行设置。

4.static getDerivedStateFromProps(nextProps, prevState):

1.在render前进行调用，触发该方法的条件是:

1.挂载时。
2.接收到新的props。
3.调用setState和forceUpdate。

2.需要return object或null。

3.在下列这些情况下不建议使用该方法:

1.执行副作用(data fetching或animation) - componentDidMount。
2.当一个props改变时需要重新计算data - memoization helper。
3.当一个props改变时不需要改变data：
    1.fully controlled。
    2.fully uncontrolled with a key。

4.不能访问this。

5.render():

1.每个组件都必须有的方法。

2.返回以下几种类型:

1.React Elements。(<div /> | <Component />)。
2.Array或Fragment。([] | <Fragment></Fragment>)
3.Number或String，渲染成Text节点。
4.Boolean或null，不渲染任何东西。
5.Portals。(详情见React API)。

6.componentDidMount():

1.在组件挂在完成后调用，可以获取DOM节点进行操作，data fetching或者animation。

2.如果订阅操作需要在ComponentWillUnmount中进行取消订阅操作。

7.shouldComponentUpdate(nextProps, nextState):

1.返回true时执行render方法，返回false时不执行render方法。

2.在此处能进行性能优化。

3.可以使用PureComponent进行替代。(shallow comparison)。

8.getSnapshotBeforeUpdate(prevProps, prevState):

1.在render之后，componentDidUpdate之前进行调用。

2.有一个返回值object，不需要设置为null，作为第三个参数传入到componentDidUpdate(prevProps, prevState, snapshot)。

9.componentWillUpdate(prevProps, prevState, snapshot):

能够在此进行操作DOM，发起服务器请求，setState，但是一定要使用if控制，否则会造成无限循环。

10.componentWillUnmount():

1.清理定时器。

2.取消网络请求。

3.清除无效的DOM元素垃圾。

Mounting:

constructor()

static getDeviedStateFromProps(nextProps, prevState)

componentDidMount()

render()

Updating:

static getDeviedStateFromProps(nextProps, prevState)

shouldComponentUpdate(nextProps, nextState)

render()

getSnapshotBeforeUpdate(prevProps, prevState)

componentDidUpdate(prevProps, prevState, snapshot)

Unmounting:

componentWillUnmount()

Error Handling:

componentDidCatch(error, info)
