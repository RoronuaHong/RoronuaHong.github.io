##关于类型检测的一些思考

整理和复习一下数据类型的知识点。

关于类型检测

JavaScript的数据类型分为2种: 基本数据类型和引用数据类型。

基本数据类型：
Number、String、Boolean、Null、Undefined、Symbol

引用数据类型:
Object

由于在开发过程中无法判断variable的数据类型，容易导致代码的执行性差。因此在开发过程中尽量将传入的variable进行判断，这样才能保证代码的质量。

类型检测的几种方法

1.typeof

第一种方式是使用typeof进行类型检测，也是最常用的方法：

通过输出可以发现：

  1. 通过typeof输出都是string类型。
  2. 只有number、boolean、undefined、string、symbol、function、object、NaN能被typeof检测正确。
  3. 对于Array、Date、RegExp、Error等实例对象也是输出object.
  4. null也为object.

因此我们在复杂的情况下无法只使用typeof完成类型检测，使用”typeof”和”==”可以判断是否为undefined或者null。

2.instanceof

instanceof在MDN中的解释为检测constructor.prototype是否存在于实例对象的原型链上。因此该方法只适用于检测构造函数与实例对象之间的关系。

因此得出以下结论：

  1. 当函数的prototype属性没有改变时,其实例化对象通过instanceof检测都为true。
  2. 当函数的prototype属性改变时，原先的实例化对象的原型链不存在了，因此检测为false；但是重新实例化的时候，检测为true。
  3. 即使将函数改变的prototype的constructor指向回该函数，通过instanceof检测还是为false。
  4. 无论是对象还是函数，其原型链中都存在在Object中，因此instanceof检测为true。

所以当我们使用instanceof进行检测的时候，被检测的variable必须是Object、Function, 但是不适合进行类型检测。

3.Object.prototype.toString.call

判断某个对象属于哪种内置属性。
调用toString方法，会将内置对象[[Class]]中的值返回出来。[[Class]]存储了一个表明该对象类型的字符串。

由上可以看出，除了NaN会被检测为Number之外，其他的数据类型都是正常的，因此使用该方法进行类型检测比较靠谱。

通过slice和toLowerCase将其转成小写的字符串即可。
